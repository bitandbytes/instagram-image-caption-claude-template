# Instagram Caption Project — @street.story.life

## About This Project
This folder contains street photography images and their corresponding Instagram caption files. The goal is to maintain a consistent caption voice and hashtag strategy across all posts.

**Photographer handle:** @street.story.life
**Camera:** Fujifilm (DSCF naming convention)
**Style:** Experimental street photography — tilted/rotated perspectives, inverted frames, human-architecture tension, both B&W and color

---

## Caption Rules

**Tone:** Poetic / minimal. Short, evocative, abstract.

**The core rule:** Do NOT describe what is in the photo. Use the visual as a springboard for a larger, more universal idea. The caption should feel like it could belong to many images, yet feel inevitable for this one.

**Good example (DSCF3503):**
> The gap between walls
> is where the city tells the truth.

This doesn't say "I shot through an underpass" — it uses the gap as a metaphor.

**Length:** 2–3 lines maximum. No full sentences required. Line breaks are intentional.

**Signature line:** Always close with `— @street.story.life` on its own line, before the hashtags.

**Always include `#SPiCollective` and `#streetphotographyinternational` in every post.**

### SPI Themed Tags

These are subcategory tags used by the @streetphotographyinternational / @spi_collective community. Using the right one increases the chance of being featured in a themed selection. Pick 1–2 per post based on what the image is strongest in:

| Tag | Use when the image features |
|---|---|
| `#spi_geometry` | Strong geometric shapes, patterns, lines, architectural symmetry |
| `#spi_street` | Classic street moments, human life in urban space |
| `#spi_life` | Candid human emotion, everyday life, fleeting moments |

When in doubt between tags, prefer `#spi_geometry` for architecture-led shots and `#spi_life` for human-led shots.

## Location Hashtag Strategy

Always add 5–7 location-specific tags based on where the photo was taken. Include the city, country, and any relevant local community tags. If the location is unknown, ask before writing the hashtag block.

Examples of the pattern to follow:
- City tag (e.g. `#bruges`, `#amsterdam`, `#tokyo`)
- Country tag (e.g. `#belgium`, `#netherlands`, `#japan`)
- Visit/tourism tag (e.g. `#visitbruges`, `#visitnetherlands`)
- Local photography community tag if one exists (e.g. `#belgium_photography`, `#dutchphotography`)
- Regional or neighbourhood tag when relevant (e.g. `#zaandam`, `#zaanseschans`)

## Folder Structure Convention

```
[root folder]/
├── CLAUDE.md
├── IMAGE.JPG + IMAGE.txt              ← single photo posts: one .txt per image, named after it
├── IMAGE.JPG + IMAGE.txt
│
├── Group 1/                          ← multi-photo post: all images + one combined .txt
│   ├── IMAGE_A.JPG
│   ├── IMAGE_B.JPG
│   └── IMAGE_A_IMAGE_B.txt           ← named after all images in the group, joined with _
│
└── Group 2/                          ← another multi-photo post
    ├── IMAGE_A.JPG
    ├── IMAGE_B.JPG
    └── IMAGE_A_IMAGE_B.txt
```

**Rules:**
- Photos in the **root folder** = individual Instagram posts, one `.txt` per image named after the image file
- Photos in a **Group X folder** = a single Instagram post (carousel/collection), one `.txt` named after all image files in the folder joined with `_` (e.g. `DSCF3955_DSCF3959.txt`)
- Caption files always live alongside their images

---

## Caption File Format

```
# FILENAME.JPG

[2–3 line abstract caption]

— @street.story.life

#tag1 #tag2 ... (up to 30, all on one line)
```

For group posts, the heading lists all image filenames and the file is named to match: `# FILE1.JPG + FILE2.JPG` → saved as `FILE1_FILE2.txt`
